# 쇼핑몰 Phase 4 구현 완료 보고서 (Walkthrough)

주문서 작성 단계의 편의성 개선('주문자와 동일' 체크), 결제 후 오입력 방지를 위한 배송 정보 2차 재확인 영역 강화, 그리고 회원 및 비회원이 공통으로 사용할 수 있는 **장바구니(Cart) 기능** 및 **주문 내역 조회(Order History) 기능**을 아우르는 **Phase 4(장바구니 & 주문내역 조회 및 사용자 경험 고도화)** 단계가 성공적으로 완료되었습니다.

---

## 1. 구현 완료 사항 요약

### 1.1. 주문서 편의 기능 및 2차 확인 강화 (Task 6)
* **'주문자와 동일' 연동**: 주문서 작성 페이지(`/orders/checkout`)에서 '주문자와 동일' 체크박스를 제공하여, 체크 시 주문자의 이름과 연락처 데이터가 수령인 입력 폼에 실시간으로 동기화 및 복사되도록 구현했습니다.
* **배송 정보 2차 재확인 영역 강화**: 주문 완료 페이지(`/orders/success`)의 배송 및 연락처 요약 영역 디자인을 대폭 개편하여, 사용자가 잘못된 주소로 주문하지 않았는지 오배송 방지용 시각적 경고 카드 및 2차 확인 레이아웃을 제공합니다.

### 1.2. 장바구니(Cart) 기능 구현 (Task 7)
* **회원/비회원 공통 로컬 스토리지 활용**: 별도 로그인 없이도 동작할 수 있도록 로컬 스토리지(`cart_items`)를 활용해 상품, 옵션, 수량 데이터를 유기적으로 보관합니다.
* **장바구니 담기 및 실시간 배지**: 상품 상세 페이지에서 '장바구니 담기' 클릭 시 카트에 추가되며, 헤더 컴포넌트(`Header.tsx`)의 장바구니 아이콘 우측에 실시간으로 총 수량이 반영되는 배지 UI를 구현했습니다.
* **장바구니 관리 및 다중 주문**: 장바구니 페이지(`/cart`)에서 담긴 상품의 체크박스 선택 제어, 수량 변경 및 실시간 합계 금액 계산, 개별/선택 삭제를 지원합니다.
* **다중 품목 주문 연동**: 장바구니에서 여러 개의 상품을 선택해 주문서로 넘길 수 있도록, 로컬 스토리지의 단일 객체 직렬화 키였던 `checkout_item`을 배열 형태의 `checkout_items`로 전환하여 바로구매(1개)와 장바구니 일괄 결제(N개) 모두에 유연하게 호환시켰습니다.
* **장바구니 버튼 레이아웃 개선 및 아이콘화**: 상품 상세 페이지에서 장바구니 버튼의 시인성을 개선하기 위해, 글자 대신 직관적인 SVG 쇼핑 카트 아이콘을 탑재하고 고정폭으로 배치하였으며, 바로구매 버튼이 가로 여백을 유연하게 채워 시각적 계층 구조가 부각되도록 레이아웃을 고도화했습니다(모바일 플로팅 영역도 동일 적용).

### 1.3. 주문 내역(Order History) 조회 기능 구현 (Task 8)
* **비회원 조회 인증 API**: 비회원 사용자가 주문 당시 입력했던 주문자명과 연락처(휴대폰 번호) 정보를 통해 자신의 주문을 조회할 수 있도록 `/api/orders/track` POST API를 신설했습니다.
* **주문 내역 조회 화면**: `/orders/history`에서 회원은 자신의 주문 목록을 즉시 볼 수 있으며, 비회원은 이름/연락처 입력을 통해 인증 성공 시 과거 주문 내역을 볼 수 있습니다.
* **아코디언 토글 레이아웃**: 주문 내역 페이지에서 각 주문 건은 아코디언 토글 형식으로 구현되어, 클릭 시 해당 주문에 포함된 상세 상품 내역(상품명, 옵션, 단가, 수량, 소계)과 수령인 배송 정보가 깔끔하게 노출됩니다.

### 1.4. 관리자 및 일반 사용자 도메인 격리 (Task 10)
* **경로 접근 통제 가드 (Middleware Guard)**: 일반 서비스 호스트(`CLIENT_HOST`)로 들어온 요청이 `/admin` 또는 `/api/admin` 하위 경로에 진입을 시도할 경우, 로그인 상태에 무관하게 즉시 `404 Not Found` 처리로 자원을 완벽히 차단 및 은폐합니다.
* **관리자 링크 조건부 렌더링**: 일반 클라이언트 도메인 접속 시에는 네비게이션 공통 헤더에서 '관리자' 텍스트 및 이동 링크 자체를 노출하지 않음으로써 사용상의 격리 및 시각적 보안을 강화했습니다.
* **다중 도메인 환경 설정 가이드**: 로컬 다중 도메인(`*.localhost`) 테스트 설정 및 프로덕션 환경배포 설정, 시나리오별 보안 동작을 문서화한 배포 가이드(`deployment_test_guide.md`)를 신설하여 보관하였습니다.

## 3. 로컬 브라우저 수동 검증 가이드

로컬 개발 서버(`npm run dev`)가 백그라운드에서 구동 중입니다. (포트: `3000`)

### 3.1. 장바구니 담기 및 다중 주문 테스트
1. 브라우저로 **[http://localhost:3000](http://localhost:3000)**에 접속합니다.
2. 상품 상세 페이지에서 옵션을 선택하고 **'장바구니 담기'**를 클릭합니다.
3. 상단 헤더의 **장바구니 아이콘 배지 수량**이 정상적으로 증가하는지 확인합니다.
4. 장바구니 페이지(`/cart`)로 이동하여 수량을 변경해 보고 총액이 실시간 반영되는지 확인합니다.
5. 장바구니 목록에서 여러 개의 상품을 선택한 상태로 **'선택 상품 주문하기'** 또는 **'전체 주문하기'**를 클릭하여 주문서 페이지(`/orders/checkout`)에 모든 상품이 정상 노출되는지 확인합니다.

### 3.2. 주문자 정보 동기화 및 2차 정보 확인 테스트
1. 주문서 작성 페이지에서 주문자명과 연락처를 입력합니다.
2. 배송 정보 카드에서 **'주문자와 동일'** 체크박스를 체크하여 수령인명과 연락처로 데이터가 복사되는지 확인합니다.
3. 결제를 진행하여 성공 페이지(`/orders/success`)로 이동한 후, 화면 하단에 표시되는 **배송 정보 2차 재확인 영역**에서 주소와 수령인 정보가 큼직하게 노출되는지 시각적으로 검증합니다.

### 3.3. 주문 내역(Order History) 조회 테스트
1. 상단 헤더의 **'주문 내역'**을 클릭하여 `/orders/history` 페이지로 이동합니다.
2. 로그인하지 않은 비회원 상태에서, 방금 주문할 때 입력했던 **주문자명**과 **연락처**를 입력하고 **'주문 조회'**를 클릭합니다.
3. 조회 성공 시 아코디언 카드 형태로 주문 건이 노출되며, 클릭하여 **상세 상품 목록 및 배송 정보**가 아코디언 토글로 슬라이딩 노출되는지 확인합니다.
